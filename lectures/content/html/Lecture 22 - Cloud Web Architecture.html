Demystifying the cloud

  • But what is going on with those computers?

Cloud computing models

  • There are three main types of services that we consume in the cloud
  • These models can help us understand what is going on in the cloud

Quick review

  • Cloud computing is a type of Internet-based computing that provides shared computer processing resources and data to computers and other devices on demand.

So in essence

  • Some company (Amazon) own a lot of networked computers
  • That company allows other people to use their computers to run applications
  • But how do we share resources?

First thought: provide computers

  • I want to run my app
  • I ask Amazon for 5 computers
  • They give it to me and I install my app on all five computers
  • If I need more, Amazon needs to network another computer
  • What are the potential issues with this?

Second thought: provide VM's

  • To scale, Amazon would have to buy new machines
  • Changing CPU/Memory/Storage would require human intervention
  • However with VM's, can provide users with their own computing resource dynamically
  • Think VirtualBox

Second thought: provide VM's

  • Can specify CPU, Memory, and Storage usage dynamically
  • Cost effective
  • Can load several VM's on one powerful computer
  • Can create and destroy them dynamically

Cloud Computing model 1: IaaS

  • Infrastructure as a Service (IaaS), think Amazon EC2 and S3
  • You are provided full access to remote Virtual Machines
  • Up to you to install an operating system, NodeJS, SQLite, etc...

IaaS

  • You have the most flexibility with the computers that run your application
  • However, you also have to build everything yourself
  • For example:
    • create two VM and install your NodeJS app on each VM
    • another VM to install your database
    • another virtual machine to handle DNS, etc...
  • You are on your own

Cloud Computing model 2: PaaS

  • Platform as a Service (PaaS), think Heroku
  • Typically built on top of an IaaS, a PaaS provides services that developers need to rapidly build applications
  • For example:
    • instead of purchasing VM's, you can purchase a worker that can run a NodeJS application
    • purchase a VM that has database software pre-installed
    • global VM's dedicated as DNS servers for your application

PaaS

  • What's exciting about PaaS is that developers don't have to worry about operations
  • It's a job in of itself to write scripts to automate installing operating systems

Cloud computing model 3: SaaS

  • Software as a Service (SaaS) comprises of application that we use that are connected to the cloud
  • Built on top of a PaaS, or even on top of a IaaS for more efficiency
  • These are applications that leverage the cloud's dynamic property to provide a service 24/7

SaaS

  • Examples: Google Docs, Dropbox, Slack, Evernote, and so on
  • These applications utilizes PaaS and IaaS in order to build a singular application
  • Typically there is no concept of databases or web servers from a PaaS
  • Nor concept of virtual machines and computing resources from a IaaS

Main Takeaway

  • It is the availability of IaaS that makes cloud services possible
  • It is the utility of PaaS that makes building cloud services much easier
  • And SaaS are the services that we use that benefit from networked computer resources
